The ultimate goal here is collecting the put options Greeks for a specified date range. However, some initial work has to be developed prior to this. Starting here, the EuropeanOption function requires a volatility measure. The implied volatility is a measure of future variability for an underlying asset which option contract is based on. Essentially its the stocks standard deviation (\(\sigma\)), which is a necessary part of the Black-Scholes-Merton formula. With this in mind, Ive collected the implied volatility for each day in the set using map. Each daily record for option price and underlying price, dividend and risk-free rate as well as the maturity is provided for the function. This yields a new data frame with 209 daily implied volatilities.

all_option_days <- seq(1, nrow(options_price), 1)

At this stage, Ive got the necessary parameters to start constructing the P&L using Greeks. As such, Ive collected the Greeks for seven days below. Before moving on, Ive also defined the Greeks below:

  • Delta: Measures the rate of change in an option price given an increase in the underlying asset price. This is essentially a slope coefficient. Commonly, delta is used when determining the likelihood of an option being in-the-money at expiration.

  • Gamma: Masures the rate of change of Delta, which is how much an option price changes given a one-point movement in the underlying asset. Delta increases or decreases along with the underlying asset price, whereas Gamma is a constant that measures the rate of change of Delta (see table below for an example of an in-the-money call option.

  • Vega: Measures the rate of change in the implied volatility of an option or position. Implied volatility is the expected volatility of the underlying asset over the life of the option not the current or historical volatility of the asset. In particular, Vega shows traders how much an option price will change for each 1% move in implied volatility (Investopedia, 2019).

To effectively collect all the Greeks, Ive developed a function. grrek_estimated_pnl conducts all the necessary delta, gamma, and Vega calculations for P&L estimation. Further, with these values derived, the P&L calculation is made. The output is a named numeric vector with all the necessary values. The output for February 26 is shown below highlighting the estimate is within a few decimals of the actual.

greek_estimated_pnl <- function(date, t, t2){
  greeks <- spread_greeks %>%
    select(date) %>%
    slice(2:4) %>%
    t() %>%
    as.vector()
  
  delta <- greeks[1] * (options_price$UndPr[t2] - options_price$UndPr[t])
  
  gamma <- .5 * greeks[2] * (options_price$UndPr[t2] - options_price$UndPr[t]) ^ 2
  
  vega <- greeks[3] * 
    (p1425_put_history$implied_vol[t2] - p1425_put_history$implied_vol[t]) 
  
  pnl_estimate <- sum(delta, gamma, vega)
  
  pnl_actual <- options_price$P1425[t2] - options_price$P1425[t]
  
  diff <- pnl_actual - pnl_estimate
  
  pnl_greeks <- c(pnl_actual = pnl_actual,
                  pnl_estimate = pnl_estimate,
                  difference = diff,
                  delta = delta,
                  gamma = gamma,
                  vega = vega)
  
  return(pnl_greeks)
}
